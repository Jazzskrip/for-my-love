# План работ

## Фаза 1: Фундамент

### Шаг 1. Разделение на файлы
**Что:** Разнести текущий монолит (advent-calendar-a.html + advent-calendar-a.css) на файлы по плановой структуре.
**Файлы:**
- `index.html` — только HTML-разметка + подключение скриптов/стилей
- `css/style.css` — все стили из advent-calendar-a.css
- `js/calendar.js` — логика календаря (рендер карточек, открытие, прогресс)
- `js/effects.js` — фоновые эффекты (снег/сердечки, конфетти)
**Зависимости:** нет
**Можно делегировать:** да, независимая задача

### Шаг 2. Разблокировка по дате + debug-меню
**Что:** Заменить текущую логику "строго по порядку" на привязку к START_DATE (2026-05-29). Пропущенные дни можно открыть. Подключить JS-логику debug-меню (стили уже есть в CSS).
**Файлы:** `js/calendar.js`
**Спек:** `docs/features/02-date-unlock.md`
**Зависимости:** шаг 1

### Шаг 3. Обновить rewards.json
**Что:** Привести data/rewards-a.json к формату из `01-data-structure.md`. Переименовать в `data/rewards.json`. Обновить пути в JS. Пока моковые данные, но структура готова под video/game/tryAgain/animation.
**Файлы:** `data/rewards.json`, `js/calendar.js`
**Спек:** `docs/features/01-data-structure.md`
**Зависимости:** шаг 1

### Шаг 4. localStorage
**Что:** Переписать хранение прогресса на 3 ключа: `adventProgress`, `adventTryAgain`, `adventDebugDate`.
**Файлы:** `js/calendar.js`
**Спек:** `docs/features/08-localstorage.md`
**Зависимости:** шаг 1
